Security Deep Dive

Every security feature explained — what it does and how it protects you.

🔐 ECDH Key Exchange

Creates a unique shared encryption key between your browser and the CORS proxy using Elliptic Curve Diffie-Hellman (P-256).

Even if traffic is intercepted, the shared secret can't be derived. Data stays encrypted end-to-end.

🔒 AES-256-GCM Encryption

All auth headers and API responses are encrypted using AES-256-GCM — the standard used by governments and banks.

Tokens and passwords appear as random blobs in transit. Network inspectors see only ciphertext.

🔑 Encrypted Vault

Credentials stored in AES-256-GCM vault. Key derived from your PIN via PBKDF2 with 600,000 iterations.

Even with storage access, brute-forcing the 600K-iteration key derivation is computationally infeasible.

🛡 SSRF Prevention

Proxy maintains a strict allowlist. Only official Discord and Instagram domains are permitted.

Prevents proxy abuse to access internal networks or unauthorized services.

🚦 Rate Limiting

Per-IP limits: 120 req/min for API, 30/min for images, 5/min for key registration.

Protects against abuse and prevents your account from being flagged for suspicious activity.

⏲ Anti-Replay

Every request includes a timestamp. Requests older than 60 seconds are rejected.

Captured traffic can't be replayed later. Prevents session hijacking.

🗃 Zero Server Storage

No passwords, tokens, or personal data stored server-side. Only an encrypted image cache with 2-day expiry.

Even a compromised server has no credentials to steal.

📂 Open Source

All code is publicly available on GitHub for anyone to audit.

No backdoors, no hidden collection, no obscurity — full transparency.

🔒 E2E Message Encryption

Optional Discord message encryption via shared passphrase. AES-256-GCM before sending through the API.

Even Discord's servers can't read E2E-encrypted messages.
