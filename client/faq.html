Security Deep Dive

Every security feature explained — what it does and how it protects you.

🔐 ECDH Key Exchange

Creates a unique shared encryption key between your browser and the CORS proxy using Elliptic Curve Diffie-Hellman (P-256). Each user gets their own key pair.

Even if someone intercepts your network traffic, they can't derive the shared secret. Your data remains encrypted end-to-end between your browser and the proxy.

🔒 AES-256-GCM Encryption

All authentication headers (tokens, cookies, CSRF tokens) and API response bodies are encrypted using AES-256-GCM — the same standard used by governments and banks.

Your tokens and passwords appear as random encrypted blobs in transit. Network inspectors (including school MITM proxies) see only meaningless ciphertext.

🔑 Encrypted Vault (PIN Protection)

Your credentials are stored in an AES-256-GCM encrypted vault. The encryption key is derived from your PIN using PBKDF2 with 600,000 iterations and a random salt.

Even if someone accesses your browser's storage, they find only encrypted data. Without your PIN, brute-forcing the 600K-iteration key derivation is computationally infeasible.

🛡 SSRF Prevention

The proxy server maintains a strict allowlist of domains. Only official Discord and Instagram domains are permitted — all other requests are rejected.

Prevents attackers from abusing the proxy to access internal networks or unauthorized services. The proxy can only talk to the services it's designed for.

🚦 Rate Limiting

The proxy enforces per-IP request limits: 120 requests/minute for API calls, 30/minute for image loading, and 5/minute for key registration.

Prevents abuse of the proxy and protects your account from being flagged by Discord/Instagram for suspicious activity. Also stops denial-of-service attempts.

⏲ Anti-Replay Protection

Every encrypted request includes a timestamp. The proxy rejects any request where the timestamp is more than 60 seconds old.

If someone captures your encrypted network traffic, they can't replay it later — the timestamp will be expired. This prevents session hijacking attacks.

🛡 Response Integrity Verification

AES-GCM includes an authentication tag that verifies the response hasn't been modified in transit.

If anyone (including a MITM proxy) tries to alter API responses — like injecting fake messages or modifying data — the authentication tag check fails and the client discards the tampered response.

🗃 Zero Server Storage

No passwords, tokens, or personal data are stored on the proxy server. The only server-side storage is an encrypted image cache with automatic 2-day expiry.

Even if the proxy server were compromised, there are no credentials to steal. Your secrets only exist in your browser's encrypted vault.

📂 Open Source Code

The entire project — client code, worker/proxy code, and documentation — is publicly available on GitHub.

Anyone can audit the code to verify there are no backdoors, hidden data collection, or malicious behavior. Security through transparency, not obscurity.

🛡 Content Security Policy

CSP headers restrict which scripts, styles, and connections the page is allowed to make. Only trusted sources are permitted.

Prevents cross-site scripting (XSS) attacks where malicious scripts could try to steal your credentials or inject harmful content.

🔒 E2E Message Encryption

Optional end-to-end encryption for Discord messages using a shared passphrase. Messages are encrypted with AES-256-GCM before being sent through the API.

Even Discord's servers can't read E2E-encrypted messages. Only people who know the passphrase can decrypt them. Messages appear as lock emoji + encrypted text to everyone else.
